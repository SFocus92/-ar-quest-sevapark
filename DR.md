## Общее описание

**AR-Quest СеваПарк** - это WebAR-приложение для интерактивного квеста в парке развлечений. Игроки используют камеру смартфона для поиска и распознавания маркеров (фотографий объектов парка), получая подсказки и в итоге - промокод на скидку.

**Основные характеристики:**
- 7 последовательных этапов квеста
- Распознавание NFT-маркеров через камеру
- Показ 3D-объектов при обнаружении маркера
- Система подсказок и продвижения по этапам
- Финальный промокод SEVA2024AR со скидкой 25%

## Как работает приложение

### 1. Старт приложения

```
Пользователь открывает сайт
        ↓
Проверка HTTPS (обязательно для камеры)
        ↓
Показ стартовой страницы (StartPage)
        ↓
Пользователь нажимает "Начать квест"
```

### 2. Инициализация AR

```
Нажатие "Начать квест"
        ↓
Загрузка A-Frame (cdn.jsdelivr.net/aframe.io)
        ↓
Ожидание инициализации THREE.js
        ↓
Загрузка MindAR UMD (mindar-image-three.umd.prod.js)
        ↓
Инициализация камеры (getUserMedia)
        ↓
Запуск AR-сканера
```

### 3. Прохождение квеста

```
Сканирование камерой
        ↓
Обнаружение NFT-маркера (mind файл)
        ↓
handleMarkerFound() → валидация этапа
        ↓
Если верный маркер:
  - Показать 3D-объект
  - Воспроизвести звук
  - Показать подсказку для следующего этапа
  - Увеличить completedSteps
        ↓
Если неверный маркер → показать ошибку
        ↓
После 7 этапов → показать промокод
```

## Конфигурация квеста

Все настройки квеста находятся в `src/lib/quest-config.ts`:

### PARK_CONFIG

```typescript
export const PARK_CONFIG = {
  name: "СеваПарк",         // Название парка
  promoCode: "SEVA2024AR", // Промокод для финала
  discount: "25%",          // Размер скидки
  finalMessage: "Поздравляем! Вы нашли все тайные метки..."
};
```

### STEPS (Этапы квеста)

Каждый этап содержит:
- `id` - уникальный идентификатор
- `order` - номер этапа (1-7)
- `markerType` - тип маркера ('nft' или 'pattern')
- `nftDescriptor` - путь к .mind файлу маркера
- `title`, `description` - текстовый контент
- `objectType` - тип 3D-объекта ('model', 'scroll', 'chest', etc.)
- `modelUrl` - путь к .glb модели
- `animation` - анимация появления

### MESSAGES

Тексты подсказок и сообщений для игрока.

## AR-система

### MindAR Integration

Проект использует MindAR для распознавания NFT-маркеров:

1. **Подготовка маркеров:**
   - Фото объектов парка → оптимизация до 1280×720
   - Генерация .mind файлов через https://carnaux.github.io/NFT-Marker-Creator/
   - Размещение в `public/assets/nft/marker-N.mind`

2. **Загрузка библиотек:**
   - A-Frame 1.4.0 (включает THREE.js)
   - MindAR UMD версия 1.2.5

3. **Flow:**
   ```
   loadARLibs() → initMindAR() → addAnchor() для каждого маркера
                    ↓
            onTargetFound → handleMarkerFound()
   ```

### Fallback режим

Если MindAR не загружается (проблемы с CDN или камерой):
- Используется простая камера через getUserMedia
- Тап по экрану симулирует обнаружение маркера
- Позволяет протестировать логику квеста

## Управление состоянием

 Zustand store (`use-quest.ts`):

```typescript
interface QuestState {
  // Состояние квеста
  isStarted: boolean
  completedSteps: number       // 0-7
  foundMarkers: string[]
  isComplete: boolean
  
  // Текущее состояние
  currentMarker: string | null
  showingContent: boolean
  
  // Состояние камеры
  cameraReady: boolean
  cameraError: string | null
  
  // Действия
  startQuest()
  handleMarkerFound(markerId)
  handleMarkerLost()
  setCameraReady()
  setCameraError()
}
```

### Persistence

Прогресс квеста сохраняется в localStorage под ключом `ar-quest-sevapark`. Сохраняется:
- `isStarted`
- `completedSteps`
- `foundMarkers`
- `isComplete`
- `soundEnabled`

## Устранение проблем

### Ошибка "Не удалось инициализировать AR-кquest"

**Причина:** Не загрузились AR библиотеки

**Решения:**
1. Проверить интернет-соединение
2. Открыть DevTools → Console, проверить ошибки
3. Попробовать режим симуляции (тап по экрану)

### Камера не запрашивает разрешение

**Причины:**
- HTTPS не используется
- Браузер не поддерживает getUserMedia

**Решения:**
1. Использовать HTTPS URL
2. Использовать Safari на iOS
3. Проверить разрешения в настройках браузера

### Маркеры не распознаются

**Причины:**
- Не созданы .mind файлы
- Плохое освещение
- Неправильное расстояние до маркера (30-100 см)

**Решения:**
1. Сгенерировать .mind файлы через NFT Marker Creator
2. Обеспечить хорошее освещение
3. Держать камеру на правильном расстоянии

### MindAR ошибка "Cannot use import statement"

**Причина:** Используется ESM версия вместо UMD

**Решение:** Использовать `.umd.prod.js` версию
