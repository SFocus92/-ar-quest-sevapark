## 🎮 Обзор проекта

AR-Квест "СеваПарк" — это интерактивное WebAR-приложение для парков развлечений.

### Особенности:

- ✅ **7 этапов квеста** с последовательной логикой
- ✅ **NFT-маркеры** — распознавание реальных объектов (скамейки, деревья, статуи)
- ✅ **Pattern-маркеры** — классические Hiro/Kanji для тестирования
- ✅ **3D-объекты** — ключи, кристаллы, порталы, сундук с сокровищами
- ✅ **Работает в браузере** — не требует установки приложения
- ✅ **iOS и Android** — поддержка Safari и Chrome
- ✅ **Промокод в финале** — настраиваемая награда

### Игровой процесс:

1. Игрок открывает ссылку в браузере телефона
2. Разрешает доступ к камере
3. Ходит по парку, наводит камеру на объекты
4. Находит маркеры по порядку (1→2→3→4→5→6→7)
5. В финале получает промокод на скидку

## 🚀 Быстрый старт

### Требования:

- Node.js 18+ (рекомендуется 20+)
- npm или bun

### Установка:

```powershell
# 1. Распакуйте архив
Expand-Archive ar-quest-sevapark.zip -DestinationPath C:\Projects

# 2. Перейдите в папку
cd C:\Projects\ar-quest-sevapark

# 3. Установите зависимости
npm install

# 4. Запустите dev-сервер
npm run dev
```

### Откройте в браузере:

- Главная страница: `http://localhost:3000`
- Маркеры для печати: `http://localhost:3000/markers`

## 🔒 Локальное тестирование с HTTPS

**Важно:** На localhost камера работает по HTTP. Для тестирования на реальном устройстве нужен HTTPS.

### Использование ngrok:

```powershell
# 1. Запустите dev-сервер
npm run dev

# 2. В другом терминале запустите ngrok
ngrok http 3000

# 3. Скопируйте HTTPS-ссылку из вывода
# Пример: https://abc123.ngrok-free.app

# 4. Откройте эту ссылку на телефоне!
```

### Установка ngrok:

1. Скачайте с https://ngrok.com/download
2. Распакуйте в удобную папку
3. Добавьте в PATH или используйте полный путь

## 📸 Создание NFT-маркеров из фотографий

NFT (Natural Feature Tracking) позволяет распознавать реальные объекты — скамейки, деревья, статуи.

### Пошаговая инструкция:

#### Шаг 1: Подготовка фотографий

**Требования к фото:**
- Разрешение: минимум 1280×720 (оптимально 1920×1080)
- Формат: PNG или JPG
- Освещение: хорошее, без теней
- Объект: чётко виден, занимает 60-80% кадра
- Фон: контрастный, без отвлекающих элементов

**Советы:**
- Фотографируйте объект под прямыми углами
- Избегайте бликов и отражений
- Делайте несколько снимков и выберите лучший

#### Шаг 2: Создание NFT-дескриптора

**Вариант A: Онлайн-инструмент AR.js**

1. Откройте: https://ar-js-org.github.io/AR.js/three.js/examples/marker-training.html

2. Перейдите на вкладку **"NFT Marker Creator"**

3. Загрузите фотографию объекта

4. Нажмите **"Generate NFT Marker"**

5. Скачайте файлы:
   - `image.fset`
   - `image.fset3`
   - `image.iset`

6. Переименуйте файлы:
   ```
   tree.fset, tree.fset3, tree.iset (для дерева)
   bench.fset, bench.fset3, bench.iset (для скамейки)
   и т.д.
   ```

**Вариант B: Локальная генерация (Linux/Mac)**

```bash
# Установите AR.js NFT tools
git clone https://github.com/AR-js-org/NFT-Marker-Creator
cd NFT-Marker-Creator

# Создайте маркер
node app.js -i photo.jpg -o marker_name
```

#### Шаг 3: Размещение файлов

Поместите скачанные файлы в папку проекта:

```
public/assets/nft/
├── tree.fset
├── tree.fset3
├── tree.iset
├── bench.fset
├── bench.fset3
├── bench.iset
├── grotto.fset
├── grotto.fset3
├── grotto.iset
├── bridge.fset
├── bridge.fset3
├── bridge.iset
├── statue.fset
├── statue.fset3
├── statue.iset
├── lake.fset
├── lake.fset3
└── lake.iset
```

#### Шаг 4: Обновление конфигурации

Откройте `src/lib/quest-config.ts` и проверьте пути:

```typescript
{
  id: 'marker_tree',
  order: 2,
  markerType: 'nft',
  nftDescriptor: '/assets/nft/tree',  // Путь без расширения!
  // ...
}
```

### Рекомендации для каждого объекта:

| Этап | Объект | Советы по фото |
|------|--------|----------------|
| 2 | Дерево | Фотографируйте ствол с характерными особенностями (дупло, наросты) |
| 3 | Скамейка | Снимайте спинку с резьбой или уникальным узором |
| 4 | Грот | Фотографируйте вход с каменной аркой |
| 5 | Мостик | Снимайте перила с вырезанными символами |
| 6 | Статуя | Фотографируйте характерную деталь (голова дракона) |
| 7 | Озеро | Снимайте иву у воды с характерным изгибом |

## 🎨 Добавление 3D-моделей

### Бесплатные источники:

1. **Sketchfab** (фильтр "Downloadable" + "Free"):
   https://sketchfab.com/features/gltf

2. **GLTF-Sample-Models**:
   https://github.com/KhronosGroup/glTF-Sample-Models

3. **Poly Pizza** (low-poly):
   https://poly.pizza/

### Требования к моделям:

- Формат: **glTF Binary (.glb)**
- Полигоны: до 50,000 (оптимально 5,000-20,000)
- Текстуры: максимум 1024×1024
- Размер файла: до 5 MB

### Добавление модели:

1. Скачайте модель в формате .glb
2. Поместите в `public/assets/models/`
3. Обновите конфигурацию:

```typescript
{
  id: 'marker_bench',
  objectType: 'model',
  modelUrl: '/assets/models/my-model.glb',
  // ...
}
```

### Оптимизация моделей:

Онлайн: https://gltf.report/
CLI: `gltf-pipeline -i model.gltf -o model.glb --draco.compressMeshes`

## ❓ Часто задаваемые вопросы

### Q: Почему камера не работает?

**A:** Проверьте:
1. Откройте по HTTPS (не HTTP)
2. Разрешите доступ к камере в браузере
3. На iOS используйте Safari (не Chrome)

### Q: Маркеры не распознаются

**A:** Проверьте:
1. Размер маркера (минимум 10×10 см)
2. Освещение (должно быть хорошим)
3. Для NFT: качество фото объекта

### Q: На iOS белый экран

**A:**
1. Используйте Safari
2. Разрешите камеру
3. Обновите iOS до 11+

### Q: Как протестировать без маркеров?

**A:** В режиме разработки тапните по экрану для имитации обнаружения маркера.

## 🔧 Устранение неполадок

### Ошибка: "HTTPS required"

**Решение:**
- Используйте ngrok для localhost
- Задеплойте на Netlify/Vercel

### Ошибка: "Camera not found"

**Решение:**
- Проверьте наличие камеры на устройстве
- Разрешите доступ в настройках браузера
- Попробуйте другой браузер

### NFT-маркер не распознаётся

**Решение:**
1. Улучшите качество фото
2. Увеличьте контрастность
3. Фотографируйте объект под тем же углом
